Twelve Column Grid

Every Project is Easier with A CSS Grid. Create HTML Pages based on the natural scrolling order of the web. HUI's 12 Column Grid aims to give you that Graph Paper style accuracy in your layout, while being flexible and liquid enough to fit just about any screen size, orientation or DPI.

New computers, old computers, netbooks, tablets, smartphones, some feature phones. The 1140 grid fits perfectly into a 1280 monitor. On smaller monitors it becomes fluid and adapts to the width of the browser. For Mobile devices, Media Queries are used.

Layout Hierarchy:

1. Basically, each content block goes inside a 'wrapper div' denoted as: .container.
.container is a full width div that allows layouts to have a background that spans the full width of the browser. It also contains 20 px padding on either side to keep content away from the edges when it becomes fluid.

2. Inside .container is the Variable %-Width wrapper: .row
.row can then be further divided into any 12 or fewer column grid style arrangement.

3. Finally, The last column within a finished .row needs to be closed out by adding .last to the ending column. (Dont forget this!)

3 Column Liquid Grid: .fourcol [1] .fourcol [2] .fourcol .last [3]

Image Scaling

If an image is smaller than the column it is contained in, it will be displayed at its original size. If it is larger then it will be shrunk to the width of the column. Don't define height and width of images inline, it'll break the proportions if they're scaled down.

From TV to The Mobile

By tweaking the original Grid, HUI is able to template at almost any view-port. From a 400 ish pixel smart screen to a 10 foot living room UI. With media queries you can even include x2 images for the iPhone 4 Retina Display.

Before you Start Coding Info

You will probably want to decide up before you start coding what you want for the "Max-Width" on .row. This is important. If you leave it to nothing, The Grid Will scale and adapt to any screen size, no matter what the dimensions. This is perfect for a 10ft TV User Interface Application, or users with retina screens. It can however be annoying if you are trying to manage every pixel and want a tight layout. In that case you can remove the comment or add .row to your css file, as be default Row is set with no limits on width.

Facebox is a jQuery-based, Facebook-style lightbox which can display images, divs, or entire remote pages.

HUI uses a maintained Fork of Facebox ver 1.3 http://github.com/defunkt/facebox.

the Easiest way to use this plugin is done by adding one line of javascript and an additional rel="facebox" to any A HREF element. The HREF is used to match the div. like so href="#div_id".

//FaceBox One Liner. Add to Doc Ready
$('a[rel*=facebox]').facebox();

Pagination

powered by jQuery plugin jqPagination. A newer method of pagination for your web site or application. Instead of displaying a list of page numbers like traditional pagination methods jqPagination uses an interactive 'Page 1 of 5' input that, when selected, allows the user to enter their desired page number. The plugin will ensure that only valid pages can be selected, a valid request will result in the paged callback. First, previous, next and last buttons work out of the box, but are optional.


<div class="pagination">
    <a href="#" class="first" data-action="first">&laquo;</a>
    <a href="#" class="previous" data-action="previous">&lsaquo;</a>
    <input type="text" readonly="readonly" data-max-page="40" />
    <a href="#" class="next" data-action="next">&rsaquo;</a>
    <a href="#" class="last" data-action="last">&raquo;</a>
</div>

$('.pagination').jqPagination({
	    paged: function(page) {
	        // do something with the page variable
	    }
	});

    Customization

    jqPagination is built to be basic, but it does have a couple options and callbacks for customization.

    Options

    current_page - option, default: 1
    Sets the current page, can also be set using the current-page data attribute on the input element.

    max_page - option, default: null
    Sets the maximum page, if this option is left null we'll use the max-page data attribute instead, if that doesn't exist we'll assume we only have one page.

    page_string - option, default: 'Page {current_page} of {max_page}'
    The string to base the input value on, no restrictions on the string, use {current_page} and {max_page} placeholders to allow the plugin to replace the respective numbers.

    Callbacks

    paged - callback, params: page
    The paged callback is called when a valid page request has been made, the page variable simply contains the page requested.

    Methods

    option - .jqPagination('option', key [, value])

    The option method allows to get / set (when supplied with a value) the following values

    • current_page
    • max_page

    Setting these values will call their respetive internal functions, and thus will perform other actions such as updating the input text.

    HorizontalNav

    Based on sebnitu.github.com/HorizontalNav and mattkersley/Responsive-Menu.
    A horizontal navigation to fit the full width of it's container. There are no requirements to how you style the navigation and adding padding, margins or borders to any of the elements wont break the effect.
    Added benefit that it Turns any <ol> or <ul> into a <select> for mobiles and low browser sizes.
